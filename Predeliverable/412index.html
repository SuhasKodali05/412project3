Load Balancer Simulator — Project 3 Design

Flow Diagram

flowchart TD A[Start] --> B[Read inputs] B --> C[Create LoadBalancer] C --> D[Generate initial queue] D --> E[Create WebServers] E --> F{More cycles?} F -->|Yes| G[Generate new requests] G --> H[Firewall check] H --> I[Assign to idle servers] I --> J[Process one cycle] J --> K[Log events] K --> L{Cooldown == 0?} L -->|No| M[Next cycle] --> F L -->|Yes| N{Queue outside limits?} N -->|Yes| O[Add or Remove server] O --> M N -->|No| M F -->|No| P[Print summary]

Description

The program starts by reading in inputs and then creating a load balancer object. Then is the generation of the intial queue and the creation of webservers.

The next part of the system is if there is more clock cycles or not. During each clock cycle, new requests can be generated and added to the queue. The firewall stage blocks the requests with blocked IP addresses. Idle servers take requests from the queue and process them one cycle at a time.

The load balancer monitors the queue size. If the queue becomes too large, a server is added. If it becomes too small, a server is removed. A cooldown period prevents too fast of scaling changes. After the final cycle, the program prints a summary of activity.
